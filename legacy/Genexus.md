## Common Procedures

### Verify Schema

1. Open the Transaction
2. File > Edit Model
3. Click "DBMS Options"
4. Check "Data Library Name"

### Verify Table of a Transaction

1. Go to "Structure" tab
2. Click the auxiliary button on the key field
3. Cross Reference... > "Tables" section

### Verify COBOL Source Generated by Genexus 9

1. Model -> "Prototype"
2. Menu "Tools" > "Explore Model Target Path"
3. Look for `.COB` files in the opened folder

### Force Generation

1. Enable "Force Generation"
2. Enable "Detailed navigation"
3. Enable "Specify & Generate"

### Fix Compilation Issues in Evo Versions

Delete the file `generator_translations.ari`.

### Fix Lucene.NET Issue in Genexus Evo 2

1. Go to menu View > Other Tool Windows > Indexer Monitor
2. Click "Pause"

Reference: [Full Text Search in Knowledge Base](http://wiki.genexus.com/commwiki/servlet/wiki?5726,Category%3AFull+Text+Search+in+Knowledge+Base)

### Add Columns at End of Table (Genexus <= 9)

For every new project, change the property "Order attributes in file" to "No":

Menu: File > Edit Model... > Tab [Data Store] > Button [Properties]

### Commit on Exit Setting for New Reports

When creating a new report, set: Transaction integrity > Commit on exit = **No**

## SQL Queries for Genexus Evolution

### Find Object References (Genexus Evolution)

```sql
SELECT FROM_MEV.MODELENTITYVERSIONNAME AS IS_REFERENCED_BY,
       TO_MEV.MODELENTITYVERSIONNAME AS HAS_REFERENCES_TO
FROM MODELCROSSREFERENCE MCR
INNER JOIN MODELENTITYVERSION FROM_MEV
    ON MCR.FROMENTITYTYPEID = FROM_MEV.ENTITYTYPEID
    AND MCR.FROMENTITYID = FROM_MEV.ENTITYID
    AND MCR.MODELID = FROM_MEV.MODELID
INNER JOIN MODELENTITYVERSION TO_MEV
    ON MCR.TOENTITYTYPEID = TO_MEV.ENTITYTYPEID
    AND MCR.TOENTITYID = TO_MEV.ENTITYID
    AND MCR.MODELID = TO_MEV.MODELID
INNER JOIN MODEL M
    ON M.MODEL_ID = MCR.MODELID
WHERE M.MODEL_NAME = 'BRANCH_NAME'
  AND (TO_MEV.MODELENTITYVERSIONNAME = 'ObjectName'
    OR FROM_MEV.MODELENTITYVERSIONNAME = 'ObjectName')
```

### Find Modified Objects by Date (Genexus Evo)

```sql
SELECT A.ModelEntityVersionName AS Name,
       B.EntityTypeName AS Type,
       A.ModelEntityVersionDescription,
       C.OutputTimeStamp AS ModifiedDate
FROM DBO.ModelEntityVersion A
JOIN EntityType B
    ON B.EntityTypeId = A.EntityTypeId
JOIN [ModelEntityOutput] C
    ON A.ModelId = C.ModelId
    AND A.EntityTypeId = C.EntityTypeId
    AND A.EntityId = C.EntityId
    AND A.EntityVersionId = C.OutputEntityVersionId
WHERE C.OutputTimeStamp >= '2016-06-01'
  AND B.EntityTypeNamespace IN ('Objects', 'DataViews', 'SubtypeGroups')
  AND A.ModelId = 1
  AND C.OutputTypeId = 301
--AND A.ModelEntityVersionName = 'OBJECT_NAME'
```
