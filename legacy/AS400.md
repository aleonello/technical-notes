## Compile Without Generating a New Source Object

Enter `14` for compile, then press `F4`. Press `F10` for additional parameters. Under source listing options, set `*NOGEN`.

## End a Job Without Creating a Log

Enter option `4`, press `F4`. Set "How to end" to `*IMMED`, "Maximum log entries" to `0`.

## Find Which Library an Object Belongs To

```
WRKOBJ
```

Use object type prefixes:
- `T` - Transaction
- `P` - Procedure
- `R` - Report
- `W` - Work Panel
- `M` - Menu
- `H` - Web Panels

## Schedule a Recurring Job

```
ADDJOBSCDE JOB(GXPGMREF) CMD(DSPPGMREF PGM(*ALLUSR/*ALL) OUTPUT(*OUTFILE) OUTFILE(GX/GXPGMREF)) FRQ(*WEEKLY) SCDDAY(*MON *TUE *WED *THU *FRI) SCDTIME('05:00:00') USER(QSYSOPR) TEXT('DESCRIPTION')
```

Verify with:

```
WRKJOBSCDE JOB(GXPGMREF)
```

## Transfer Objects Between AS400 via FTP and SaveFile

**On source AS400:**

1. Create the Save File:

```
CRTSAVF
```

Fill in "Save file" name (e.g. `TESTESF1`) and "Library" (e.g. `GXPGMNCR`).

2. Save the object to the Save File:

```
SAVOBJ
```

Configure: Objects = `<object_name>`, Library = `<source_library>`, Device = `*SAVF`. Press `F10` for additional parameters, set save file name and library.

**Transfer via FileZilla:**

- Access the Save File path: `/QSYS.LIB/<Library>.LIB`
- Transfer the `.SAVF` file to local machine

**On destination AS400:**

1. Create Save File with the same name
2. Upload the `.SAVF` file via FileZilla to the same path
3. Restore the object:

```
RSTOBJ
```

Configure: Objects = `<object_name>`, Saved Library = `<source_library>`, Device = `*SAVF`. Press `F10`, set save file name and library.

## Transfer COBOL Source via FileZilla

Configure FileZilla site manager with `sitemanager.xml`, adding the following between `EncodingType` and `BypassProxy`:

```xml
<PostLoginCommands>
    <Command> ascii </Command>
    <Command> site namefmt 1 </Command>
    <Command> site listfmt 1 </Command>
    <Command> cwd / </Command>
</PostLoginCommands>
```

Access library at `/QSYS.LIB/<library>.LIB`, navigate to `QCLSRC.FILE`. Set transfer type to ASCII before downloading `.MBR` files.

## DB2 Queries

### Verify Objects via SQL (Programs)

```sql
SELECT * FROM TABLE(QSYS2.OBJECT_STATISTICS('<LIBRARY>', 'PGM')) AS X
WHERE OBJNAME IN ('OBJ1', 'OBJ2', 'OBJ3')
```

### Verify Objects via SQL (Tables)

```sql
SELECT * FROM TABLE(QSYS2.OBJECT_STATISTICS('<LIBRARY>', 'FILE')) AS X
WHERE OBJNAME IN ('OBJ1', 'OBJ2', 'OBJ3')
```

### Verify Column Information

```sql
SELECT TABLE_SCHEMA,
       TABLE_NAME,
       COLUMN_NAME,
       IS_NULLABLE,
       DATA_TYPE,
       CHARACTER_MAXIMUM_LENGTH,
       NUMERIC_PRECISION,
       NUMERIC_SCALE
FROM SYSIBM.COLUMNS
WHERE TABLE_SCHEMA IN ('SCHEMA_NAME')
  AND TABLE_NAME = 'TABLE_NAME'
  AND NUMERIC_SCALE > 0
ORDER BY TABLE_SCHEMA, TABLE_NAME, ORDINAL_POSITION
```

### List Stored Procedures in DB2 (excluding system)

```sql
SELECT SPECIFIC_SCHEMA, SPECIFIC_NAME
FROM QSYS2.SYSPROCS
WHERE NOT (SPECIFIC_SCHEMA IN ('QSYS', 'SYSIBM', 'SQLJ', 'EJSADMIN', 'QIWS'))
ORDER BY SPECIFIC_SCHEMA, SPECIFIC_NAME
```

## Recover Deleted Records from Journal

1. Check if table has journaling enabled:

```
DSPFD <TABLE> <LIBRARY>
```

The image type must be `*BOTH`.

2. Check for locks on the table:

```
WRKOBJLCK <TABLE>
```

End all locks with option 4 > F4 > `*IMMED`.

3. Display journal to find the delete entry (type `R DL`):

```
DSPJRN
```

Note the sequence number of the `DL` entry.

4. Remove the journal change (reverting the deletion):

```
RMVJRNCHG
```

Set the journal, table, sequence number, and `Commitment boundary = *NO`.

> Note: For updates, use `UB` (before-image) and `UP` (after-image). Set "Starting large sequence number" > "Ending large sequence number" to revert.

## Update GXPGMREF Object Reference Table

### Method 1 - Direct Command

```
DSPPGMREF PGM(*ALLUSR/*ALL) OUTPUT(*OUTFILE) OUTFILE(GXDTANCL/GXPGMREF)
```

When the message "Waiting for reply to message on message queue QSYSOPR" appears, open another session (`Shift+Esc`, enter `1`), run `DSPMSG F4`, set Message Queue to `SYSOPR`, find the message, enter `9999` and press Enter.

### Method 2 - Submit as Job

```
SBMJOB -> F4
Command: DSPPGMREF PGM(*ALLUSR/*ALL) OUTPUT(*OUTFILE) OUTFILE(GXDTANCL/GXPGMREF)
Job name: GXPGMREF
```

Monitor with `WRKACTJOB`. If status shows `MSGW`, enter option `7` to display message, type `9999` and press Enter.
